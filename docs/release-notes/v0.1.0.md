# v0.1.0 — first pre-alpha

The first usable cut of Ōtori: a local-first music player that
organizes, performs, and is safe to delegate to an AI agent. macOS
only, Apple silicon. **Pre-alpha — expect breaking changes before
1.0**; the 1.0 bar is in
[CONTRIBUTING.md § "Reaching 1.0"](../CONTRIBUTING.md).

## What works

- **Scan & index** — iCloud-aware recursive scan into a SQLite index
  with the provenance schema from day one. `.icloud` placeholders are
  skipped and reported, never mass-downloaded.
- **Playback with stage presence** — two-deck gapless engine,
  beat-matched DJ crossfades between tempo-compatible tracks, live
  48-bar spectrum (log-frequency, dB-scaled, peak-hold, 60fps),
  display-sleep blocking while Stage plays.
- **Stage / Backstage dual-mode UI** — one keystroke between a dense
  management table and a performance surface with cover-art gel
  lighting, beat-reactive visuals, and synced scrolling lyrics.
- **Tag management with a trust stack** — Swinsian-grade read/write
  over your actual files. Dry-run by default; every `--apply` is an
  undoable transaction with a first-touch snapshot. Human-curated
  fields are untouchable by agents unless you explicitly override.
  Provenance: `human` > `agent` > `import` > `inferred`.
- **Beat This! BPM analysis** — SOTA transformer (pure-Rust ONNX
  runtime) runs in-app and headless; model weights ship inside the
  `.app`. Reanalyze low-confidence tracks or the whole library.
- **Synced lyrics** — LRC parsing + scroll, word-level where data
  exists, per-track sync nudge. Sidecar `.lrc` editing in the
  inspector.
- **Cover art** — resolution chain (embedded → sidecar → folder),
  inspector cover removal, jacket fetch from VocaDB.
- **Agent-operable CLI** — the entire feature surface is drivable
  through `otori` with `--json`, stable versioned schemas, semantic
  exit codes. See [AGENTS.md](../AGENTS.md) — the operating contract.

## What doesn't (yet)

- **Smart playlists and watch folders** — roadmap cut #3 ("It
  performs") is partial (◐). Stage mode, spectrum, and synced lyrics
  ship; smart playlists and watch folders do not.
- **Other platforms** — macOS only (Apple silicon build). The core
  avoids mac-only APIs to keep portability open.
- **No auto-update** — Tauri updater is not wired. Update by
  re-downloading.
- **No streaming / recommendations / DRM** — explicit non-goals for
  1.0.

## Installing

Pre-built `Ōtori_0.1.0_aarch64.dmg` is attached to this release. The
build is **unsigned** — macOS Gatekeeper blocks it on first open.
Either right-click → *Open*, or strip the quarantine flag once:

```bash
xattr -dr com.apple.quarantine /Applications/Ōtori.app
```

Signing + notarization (Developer ID) arrives at 1.0. Building from
source needs no signing — see
[README § "Building from source"](../README.md#building-from-source).

## Known limits

- The library schema is at v13 and still moving; a future detection
  change may reopen analysis. Breaking changes between `0.x` releases
  are expected and need no deprecation cycle.
- The CLI JSON contract (`otori --schema-version`) is additive-only
  within a minor; breaking changes bump it.
